Parte 5

En esta quinta parte, lo que se va a realizar es el pago de nominas y el pago de bonificaciones, por lo que se van a crear dos funciones. La primera que se llamara pagarnomina y la segunda que se llamara pagarbonificaciones. Estas dos funciones manejan una estructura similar, por lo que al realizar una, la segunda sera mas sencilla aun de entender. Adicional a esto, se harán el llamado a las funciones en la función principal main.

¿Como se realizara?

Para pagarnomina

Igual que como el apartado anterior de eliminar empleado, se va a pedir que se ingrese la cedula, después se cojera la longitud de la lista de empleados y se recorrerá la lista por indice atreves de un ciclo for, se cojera el empleado correspondiente, y se validara que el capital de nomina actual, sea mayor que el valor de nomina del empleado. Una vez se haya validado y se cumpla esa condición el capital de nomina disminuirá con lo que se le pago al empleado, dado el caso de que no se cumpla la condición de que el capital de nomina sea mayor al sueldo del empleado usaremos un break para romper el ciclo for, se retornara un booleano True, y el capital de nomina actualizado. Dado el caso que nunca se haya encontrado un empleado con dicha cedula ingresada, entonces se retornara un False y el capital de nomina.

Para pagarbonificaciones

En esta función se hará exactamente lo mismo que en la función anterior, solo que aquí se validara que el numero de ventas del empleado escogido sea mayor a 2 y menor a 4, para así poder aplicar la bonificación del 20%. Si no se cumple esta condición se validara entonces que el numero de ventas sea mayor a 4 para poder aplicar así la bonificación del 40%. Si ninguna de estas condiciones se cumple, entonces se imprimirá por pantalla que el empleado no recibe bonificación.
